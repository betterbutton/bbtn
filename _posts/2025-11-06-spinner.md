# Spinner
> Spinner is a small visual element that indicates to the user that the system is working.

[Spinner](https://mui.com/material-ui/react-progress/) is a small animated icon, usually a rotating circle, that shows the system is processing the user's request. Its purpose is simple: to prevent the user from feeling that the application has frozen or is unresponsive. In other words, it helps maintain trust during the waiting moment.

Spinners work well when dealing with short and unpredictable processing of provided data – for example during login, saving settings, or loading content. They show that something is happening without revealing how long it will take. For such tasks, people don't need to know the exact waiting time; it's enough for them to see that the system is responding.

When waiting lasts longer than a few seconds, the spinner loses its meaning. Continuous spinning without end causes uncertainty and restlessness in the user. If a longer process is expected – for example file upload, data synchronization, or update installation – it's better to use a progress indicator, percentages, or a step overview. Such elements give the user a sense of control.

Modern interfaces often [no longer offer a full-screen spinner](https://uxdesign.cc/stop-using-a-loading-spinner-theres-something-better-d186194f771e). Instead, spinners appear in a local context – for example directly within a button that remains part of the functional interface. Other applications use so-called skeletons – gray outlines of future content that preserve the page structure and feel faster.

In fact, a spinner is not a technical tool but a psychological signal. It's not a measuring device but an assurance. Used with measure and thoughtfulness, it bridges the short gap between when a person acts and when the system responds – a silent, spinning symbol of patience and trust.
