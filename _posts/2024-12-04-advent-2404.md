# Motivate users to use secure passwords
> 

To protect user accounts, secure passwords also contribute. As website creators, we can help users create strong and secure passwords, although this option is often underestimated. Instead of complex rules and frequent password changes, websites and applications should focus on modern recommendations that reflect current standards.

# Fewer rules, more length

According to recommendations from the American [National Institute of Standards and Technology](https://pages.nist.gov/800-63-4/sp800-63b.html#appA) (NIST), passwords should be:

- **Long**: A minimum length of 8 characters is increasingly insufficient as a baseline. Motivate users with a password length meter to enter passwords of at least 15 characters. Allow up to 64 characters (or more), enabling users to leverage password managers and create truly strong passwords.

- **Without complex rules**: Requirements for special characters or combinations of uppercase and lowercase letters often lead users to recycle or write down passwords. The only rule should be length.

# Recovering forgotten passwords – increase security

Recovering a forgotten password must be simple but secure. The process should involve user authentication but must not be based on questions like *"What was your first pet's name?"* These methods are easily exploitable. Instead, use email verification, SMS, or multi-factor authentication.

# Forget about regular password changes

Frequent password changes lead to simplification or repetition. Therefore, do not enforce regular password changes on users; change them only if there is suspicion of a breach or compromise.

# Verify passwords against the database of leaked passwords

The [Have I Been Pwned](https://haveibeenpwned.com/) project offers an updated [database of 320 million unique cracked passwords](https://www.troyhunt.com/introducing-306-million-freely-downloadable-pwned-passwords/) [Pwned Passwords](https://github.com/HaveIBeenPwned/PwnedPasswordsDownloader) for download. These passwords, hashed for privacy, can be used to verify whether users are using compromised combinations. The project highlights the risk of reusing cracked passwords and helps organizations increase their users' security.

Users will appreciate the ease of use, and you will increase security without unnecessary complications. Your website or application will thus be one step closer to the ideal of a secure and user-friendly environment.🎄04
