# Miller's Law
> A principle that highlights the limited capacity of working memory and helps designers simplify user interfaces.

**Miller's Law**, sometimes also Miller's Law ([Miller's Law](https://lawsofux.com/millers-law/)) or [Miller's Magic Number](https://cs.wikipedia.org/wiki/Millerovo_magick%C3%A9_%C4%8D%C3%ADslo), says that a person can hold about seven things in memory. In user interface design, this means we shouldn't present too many options at once—especially not if the user has to remember them. If content is divided into smaller parts, people process it more easily.

The rule [comes from research](https://web.archive.org/web/20090326223523/http://www2.psych.utoronto.ca/users/peterson/psy430s2001/Miller%20GA%20Magical%20Seven%20Psych%20Review%201955.pdf) by psychologist [George Miller](https://www.reknisioweb.cz/p/george-miller) from the 1950s. It is often used in web and app design. However, it is not an exact number, but rather a reminder that the brain has its limits. We overcome the limitation of the number seven through chunking (dividing into parts), recoding, using more categories or dimensions, or gradual processing.

Designers, according to Miller's Law, divide complex forms into steps, simplify menus, or group information. For example, in a mobile app, there should not be too many buttons on one screen.

Miller's Law is, however, just one tool. A good interface also depends on other factors—for example, how quickly the user decides, how clear the interface is, and how much they have to think when using it.

Modern design strives to be considerate. It's not about adhering to a number, but about people feeling comfortable in the environment. If an interface is to match human thinking, it should be as simple and understandable as possible.

Properly used, Miller's Law helps create interfaces that are pleasant and easy to use.
