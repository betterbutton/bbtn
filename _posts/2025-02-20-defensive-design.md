# Defensive Design
> Defensive design anticipates errors, makes them easier to understand, and allows for easy recovery, thereby improving the user experience.

Defensive or **defensive design** ([defensive design](https://www.smashingmagazine.com/2011/05/getting-started-with-defensive-web-design/)) is an approach to interface design that minimizes the occurrence and impact of user errors and reduces frustration. In the book [Defensive Design for the Web](https://www.goodreads.com/book/show/104752.Defensive_Design_for_the_Web), Jason Fried and Matthew Linderman from 37signals show how systems can help users handle crisis situations and navigate the digital environment without unnecessary obstacles.

When Fried and Linderman were writing their book, it was common to accidentally send an email without an attachment, and not only would your mail program fail to warn you, but it also wouldn't give you the option to recall the sent email. It was a time when we subconsciously pressed the Ctrl-S or Cmd-S keyboard shortcut every few moments, because changes weren't saved automatically.

Defensive design follows ten simple principles:

1.  It's good to prevent errors.
2.  If an error does occur, the user receives clear information about where the problem is.
3.  The system offers a way to recover from the consequences of errors.
4.  Navigation clearly shows where we are and what our next options are.
5.  Default values are set meaningfully.
6.  The system signals what state it is in.
7.  A user in trouble is provided with help.
8.  Irreversible actions require conscious confirmation and a way back.
9.  Every user action is followed by a clear system response.
10. In critical situations, the system offers active help and assistance.

Defensive design helps designers create intuitive and reliable environments. This approach increases user trust, minimizes errors, and reduces customer support costs.
