# Responsive design
> Designing for various screen sizes using flexible layouts, media queries, and web standards for an accessible and high-performance user experience.

[Responsive web design](https://www.vzhurudolu.cz/prirucka/3-principy-rwd) ([responsive web design](https://developer.mozilla.org/en-US/docs/Learn_web_development/Core/CSS_layout/Responsive_Design), RWD) ensures that websites and applications function correctly across devices. Whether it's a mobile phone, tablet, or large monitor, the user should always have a clear and comfortable experience.

Today, it represents a [common standard](https://web.dev/learn/design) that influences not only appearance but also accessibility, loading speed, and search engine ranking. The foundation consists of flexible layouts, images that adapt to the available space, and code rules that determine what to display at different screen sizes.

Developers often start with a mobile-first design and only then expand the layout for larger screens. It's not just about fitting the page on the screen. It must be readable, easily navigable, and quick to load. If it doesn't work well, users leave and search engines take notice.

The concept of responsive design was first described in 2010 in Ethan Marcotte's article [Responsive Web Design](https://alistapart.com/article/responsive-web-design/) on A List Apart. Since then, it has been constantly evolving. Today, we talk about, for example, more precise control over individual interface components.

Responsive design is not just a technical solution. It's an approach that says: every user, on any device, deserves an equally high-quality experience.
