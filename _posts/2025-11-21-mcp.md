# Model context protocol
> MCP is an open standard connecting artificial intelligence with tools and data. It enables smarter, context-aware workflows in design and development.

The Model Context Protocol, or MCP ([model context protocol](https://modelcontextprotocol.io/)), is a universal interface for connecting tools to artificial intelligence. It allows systems like [GPT](https://platform.openai.com/docs/mcp) or [Claude](https://www.anthropic.com/news/model-context-protocol) to communicate securely with external services—such as [Figma](https://help.figma.com/hc/en-us/articles/32132100833559-Guide-to-the-Figma-MCP-server), [Notion](https://developers.notion.com/docs/mcp), or [GitHub](https://github.com/github/github-mcp-server). It was created by Anthropic in 2024. MCP uses a client-server architecture and gives AI access to tools, data sources, and pre-built task templates through a single, common interface.

MCP has changed the way digital teams work. UX designers can generate code directly from designs, verify accessibility against standards, and have documentation created automatically. MCP-based tools save time and accelerate development.

For teams building websites or applications, this means faster prototyping, smarter design handoffs, and AI assistants that understand the context. They can pull in customer feedback from support, discover component parameters from design systems—all in real time.

MCP is an [open standard](https://github.com/modelcontextprotocol), so anyone can connect their own tool. And because it is not tied to a single vendor, it doesn't lock you into any ecosystem.

Its implementation, however, requires a thoughtful approach—especially when setting permissions and defining what the AI can do. But when done right, MCP transforms artificial intelligence from a tool into a true collaborator that understands your work and can build upon it.
