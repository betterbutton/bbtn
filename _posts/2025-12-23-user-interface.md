# User Interface
> User interface is a set of structures, controls, and feedback that enable people to use and understand what is happening on a website.

[User Interface](https://wikisofia.cz/wiki/U%C5%BEivatelsk%C3%A9_rozhran%C3%AD) ([user interface](https://en.wikipedia.org/wiki/User_interface)) is a website's workspace. This is where users decide what to click, what to do next, and whether their action had the expected result. For website owners and managers, interface quality directly impacts task completion, error rates, support costs, search engine findability, and user trust. When people encounter difficulties, it's usually not because a feature is missing, but because the interface poorly exposes or explains it.

In practice, a user interface consists of navigation, page structure, interactive elements, and feedback after every action. Clear labels, predictable behavior, and visible system status allow users to act with confidence. Once feedback is missing or unclear, users hesitate, repeat actions, or abandon the task entirely.

User interface is often confused with user experience. Experience describes the overall perception of a service over time. Interface is narrower and more specific. It's the layer that translates user intent into system behavior and back. Without a usable interface, a good experience cannot emerge, regardless of content quality or background technology.

Decades of usability research agree on several stable principles. An interface should make available options visible, use the user's language, favor recognition over recall, provide immediate feedback, and prevent errors or make them easy to correct. These principles recur in the work of Jakob Nielsen and Ben Shneiderman, and in international standards.

For website managers, user interface isn't a one-time design phase but an ongoing responsibility. As a website grows, features accumulate, and devices change, the interface tends to become more complex. Regularly verifying key tasks, feedback states, accessibility, and responsiveness helps keep the website usable in daily reality, not just on paper.
