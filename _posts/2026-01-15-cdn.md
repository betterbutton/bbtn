# Content delivery network (CDN)
> A distributed network of servers that delivers web content from locations closer to users to speed up response times, increase reliability, and reduce the load on the origin server.

*[Content delivery network](<<<URL_0_a4a959>>)*, or CDN, acts as an intermediate layer between the user and the origin server. It is a distributed cache. It takes on a significant portion of the work that would otherwise have to be handled by a single location. The result is faster loading, higher resilience during spikes, and more stable website behavior across regions.

From a technical standpoint, a CDN routes requests to the nearest suitable server. If the response is available in the cache, it is returned immediately. If not, the CDN forwards the request to the origin server and stores the response for future use. The origin server remains the source of truth, but it is not burdened by every single request. It is precisely the reduction in distance and repeated transfers that lowers latency and infrastructure demands.

The cache is only effective, however, when rules are clearly defined. A CDN follows standard HTTP protocol rules and relies on responses to unambiguously state whether and how long they can be stored. For static files, this enables long-term caching and significant acceleration. For dynamic content, greater caution is required, as ambiguity multiplies quickly in a distributed environment.

Modern CDNs also optimize the data transfer itself. They terminate encrypted connections at the network edge, support modern transfer protocols, and improve connection behavior even on unstable networks. The same infrastructure today also serves a security role. Distributed capacity helps manage attacks, and request filtering happens before they reach the origin server. Thus, a CDN is not just an acceleration tool, but a layer that makes a website more predictable and resilient against the realities of operation.
