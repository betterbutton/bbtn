# A user interface is like a joke
> A design principle stating that a good interface – like a good joke – should be instantly understandable without explanation.

> *You are opening the fourth window of the BBtn Advent Calendar for 2025. In this year's advent series, we will look at popular UX designer proverbs. Today's one is: A user interface is like a joke. If you have to explain it, it’s not that good.*

[A user interface is like a joke.](https://x.com/martinleblanc/status/466638260195041280)

*"A user interface is like a joke. If you have to explain it, it’s not that good."* This bon mot was written in 2014 by designer *[Martin LeBlanc](https://www.linkedin.com/in/martinleblanc/)*. The saying became popular and turned into one of the most used proverbs about usability. In a single phrase, it summarizes what user experience experts have been trying to do for decades – make understanding easier.

The essence of the idea is simplicity and immediate comprehensibility. *[Steve Krug](https://sensible.com/about/)* in the book *[Don't Make Me Think](https://www.goodreads.com/book/show/18197267-don-t-make-me-think-revisited)* says that a user should not have to stop and think about how to use a website. *[Don Norman](https://www.nngroup.com/people/don-norman/)* described the [gulf of execution and evaluation](https://www.reknisioweb.cz/p/gulf-of-execution-and-evaluation) – the effort we must exert to figure out what to do and what happened. *[Jakob Nielsen](https://www.nngroup.com/people/jakob-nielsen/)* formulated the principles that follow from this: speak the user's language, be consistent, provide clear feedback. They all aim for the same thing – for the interface to explain itself.

If a user has to look for help, read instructions, or guess what an icon means, the design has failed. Familiar patterns, understandable labels, and visible elements reduce mental effort and allow users to focus on their goal, not the interface.

However, simplicity has its limits. More complex tools, professional applications, or accessible interfaces for users with specific needs often require explanation. In such cases, help is not a failure but a sign of consideration. Balance is key – basic tasks must be self-explanatory, and supporting information should only help where needed.

For web designers and site managers, Martin's bon mot is both a warning and a compass. If people often ask how to do something that seems obvious, the interface is the joke that didn't land. Adjust, simplify, test again – until users understand without explanation.
