# Test your website in slow connection mode
> 

Website or mobile app loading speed affects user satisfaction. You might be working on a fast connection, but many users have worse conditions—for example, they use a cheap mobile data plan, travel to remote areas, or live in places with poor coverage. Therefore, it's important to test how your website behaves in slow connection mode. This will help you uncover issues and ensure a better user experience for these visitors as well.

## How to test a website on a slow connection?

Testing a website is simple thanks to tools that allow you to simulate various internet speeds. The fastest option is using developer tools in browsers.

If you use Google Chrome, open the developer tools using the keyboard shortcut `F12` or `Command/Ctrl-Shift-I`. Go to the Network tab. In the Throttling settings, select, for example, Slow 3G or Fast 3G. Then reload the page and observe how long the content takes to load and in what order it appears.

Similar features can be found in the Firefox browser, where you can also limit connection speed.

If you want a more comprehensive test, try online tools like [WebPageTest](https://www.webpagetest.org/) or [Lighthouse](https://developers.google.com/web/tools/lighthouse/), which is part of Chrome DevTools. These tools simulate a slow connection and simultaneously generate a clear report with specific recommendations for performance improvement.

## What to focus on during testing?

When testing a website on a slow connection, it's important to monitor several key points.

**First, page load time.** Focus on metrics like First Contentful Paint, which is the time when some content first appears, and Largest Contentful Paint, which measures the time to load the largest portion of content.

**Second, file sizes.** Large images, videos, or unoptimized scripts can significantly slow down loading.

**Third, the number of requests.** The more requests a website sends to the server, the slower its loading can be. You can solve this problem, for example, by merging CSS and JavaScript files or minifying them.

## Tips for performance improvement

Optimizing a website for slow connections isn't complicated. The first step is [image compression and optimization](https://www.reknisioweb.cz/p/advent-2401-obrazky). Use modern formats like WebP or AVIF and minimize file sizes.

Another important element is [lazy loading](https://www.reknisioweb.cz/p/advent-2417). This approach ensures that images and other content load gradually, only when they are actually needed.

If you have content stored on a server, consider implementing a Content Delivery Network, or CDN. This technology helps speed up loading by serving files from the nearest available server.

## Why is this important?

Testing your website in slow connection mode allows you to [create a faster and more accessible website](https://docs.pagespeed.cz/docs/why-page-speed). The result will be not only a better user experience but also a lower bounce rate and higher visitor satisfaction.

A fast and optimized website is now considered a given by users—regardless of how fast their current connection is.🎄18
