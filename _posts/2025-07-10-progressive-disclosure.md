# Progressive Disclosure
> User interfaces that offer simplicity first and complexity later—showing people only what they truly need, exactly when they need it.

**Progressive Disclosure** ([progressive disclosure](https://www.interaction-design.org/literature/topics/progressive-disclosure)) is a design principle that helps people better navigate apps or websites. Instead of showing users all options at once, only the most important elements appear first. Additional features are revealed only when the user asks for them—for example, by clicking a *"Advanced settings"* button.

This reduces mental load and increases the chance that users won't get lost in the interface. Beginners appreciate the clarity, while experienced users gain the ability to dive deeper.

A typical example might be a website management dashboard. If it displayed all statistics, toggles, and graphs immediately, it would be hard to navigate. A well-designed interface would first show only a few key metrics and offer further details, perhaps after a click.

The success of this principle depends on correctly estimating what the user currently needs. If something important is hidden too deep, it won't work. If, on the other hand, everything is shown at once, it overwhelms attention.

When done well, progressive disclosure improves both clarity and usability. It also sends users a clear message: we understand what you need—and we'll show you more when the time comes.
