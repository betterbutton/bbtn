# Graceful degradation and progressive enhancement
> Graceful degradation and progressive enhancement: Two web approaches. From modern features with limited support to a universal foundation with advanced improvements.

[Graceful degradation](https://zdrojak.cz/clanky/graceful-degradation-vs-progressive-enhancement/) (graceful degradation) relies on the full version for modern browsers and only then addresses how to adapt to less capable devices. Sometimes, however, older environments are forgotten to be tested, leaving users with slow internet without access to content.

[Progressive enhancement](https://developer.mozilla.org/en-US/docs/Glossary/Graceful_degradation) (progressive enhancement) builds from the bottom up. First, a simple, universally functional foundation is created that can manage with minimal technology. For better phones and browsers, advanced features are then gradually added. This ensures everyone has at least basic access to information, while those with the latest hardware get bonus extras.

For complex applications, [graceful degradation](https://developer.mozilla.org/en-US/docs/Glossary/Graceful_degradation) is sometimes used because it is fast for prototyping. When implemented correctly, it can work well, but if you forget tests, unexpected problems can arise. On the other hand, [progressive enhancement](https://developer.mozilla.org/en-US/docs/Glossary/Progressive_Enhancement) requires thoughtful design from the start, which is suitable for blogs or news sites, for example. Thanks to this, people with any device get access to the same content, which search engines also appreciate.

If you don't want to waste users' time, think about them from the very beginning. Build the web so that the basic layer works everywhere, and only then gradually add further improvements. This approach almost always pays off in the long run.
