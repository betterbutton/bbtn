# Single-page applications
> A web application that loads once and dynamically updates content without reloading the entire page.

[Single-page applications](https://learn.microsoft.com/cs-cz/entra/identity-platform/scenario-spa-app-configuration) ([single page applications](https://en.wikipedia.org/wiki/Single-page_application), SPA) load only one HTML page upon first opening and then dynamically change content without needing to reload. The result is a smooth, fast, and natural movement within the interface, reminiscent of working with a native application. For designers and developers, this means fewer disruptive moments, faster response times, and greater control over how the user navigates the application.

While traditional websites load a new page from the server with every click, SPAs use JavaScript to intercept the interaction, fetch data in the background, and modify only what is necessary. This makes the entire experience feel much more natural and faster.

This architecture uses modern tools like React, Vue, Angular, or Svelte. These allow the interface to be divided into reusable components, easily manage application state, and navigate between views without page transitions. Communication with the server occurs via REST or GraphQL APIs.

But there's another side to the coin. Because SPAs rely on JavaScript, they can encounter issues with accessibility or search engine optimization. The initial load is heavier and requires smart strategies for code minification and caching. Nevertheless, they remain the best choice for applications requiring rich and frequent interaction.

SPAs change how we design for the web – instead of page switching, we design state transitions. When done well, the user barely notices the difference. And that's the point.
