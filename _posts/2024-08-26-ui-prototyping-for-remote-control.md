# UI Prototyping with Remote Control
> 

When you design an app for a smart TV, users will control it with a remote. It can be useful to have the ability to create a prototype for the TV and test it specifically with a remote control.

You can create the prototype as you usually would, for example in Figma. You connect the TV to the computer as a display. And now you just need to solve the remote control issue. Basically, there are two options: [infrared remote and Bluetooth remote](https://www.amazon.de/hz/wishlist/ls/OIPGRNN0DSDC).

Jiří Martínek from 2Fresh recommends the Flirc infrared signal receiver, which [he uses for testing prototypes for O2 TV](https://www.linkedin.com/feed/update/urn:li:activity:7229788926616489984/). You plug the receiver into the computer's USB port and pair it with any standard infrared remote. Physics, of course, applies in this case too, so the remote should ideally be aimed at the signal receiver. You should therefore ideally position the computer so that it can receive the signal when the user aims the remote at the TV.

With Bluetooth remotes, direct line of sight is not necessary, and if you already have Bluetooth on your computer, you don't need a receiver. You simply pair the remote with the computer, just like pairing a mouse or presenter, and you're done.

In the prototype, you then just map the activation of an action to a button press on the remote, and the user won't even notice they are working with a prototype and not a real application.
