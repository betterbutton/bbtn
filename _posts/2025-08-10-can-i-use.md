# Can I Use: Instant Browser Compatibility Check
> Can I Use is a web tool that clearly shows current support for modern front-end features in browsers.

![Screenshot of the Can I Use service's home screen.](https://substack-post-media.s3.amazonaws.com/public/images/ba5d352a-ea0f-49b8-a41f-6336dba4303e_1895x1073.png "610x345")

[Can I Use](https://caniuse.com) knows which website features browsers support.

If you're a web developer deciding whether to use, say, CSS Grid, WebP images, or a new JavaScript feature, [Can I Use](https://caniuse.com) will tell you within seconds. Just enter the feature name and you'll see a color-coded table of support across browsers, from desktop to mobile, including notes on partial support and ways to supplement it.

Data is regularly updated according to the rapid development of browsers and, in addition to the website, is often available in developer tools, such as a VS Code extension, or as a checklist when building a project. This allows you to uncover compatibility issues even before launching the website.

It's not just a technical aid. The page also plays a strategic role. It allows you to plan gradual improvements, justify or reject certain features, and estimate when it will be safe to use them for everyone. You see specifically who supports the feature now, who will add it soon and where the gaps are.

The interface is intentionally simple. You type the feature and get clear facts. You can go deeper and look at implementation details or related specifications. Thanks to this clarity, you avoid the "compatibility trap" – a situation where part of your users don't see the feature at all.

In an industry where "works everywhere" determines user trust and business success, [Can I Use](https://caniuse.com) not only provides information but also a solid foundation for better design and development.
