# Marek Prokop advises what to do when you don't need a page in search results
> 

Unnecessary pages in search results can unnecessarily occupy the search engine robot, which then doesn't get to the pages that matter. The best practice is not to link to these pages at all in a way visible to search engines, [advises SEO specialist Marek Prokop](https://www.linkedin.com/feed/update/urn:li:activity:7263156396886577152/). This can be solved, for example, with JavaScript (ideally loading from `` a URL that is forbidden in [robots.txt](https://www.jakpsatweb.cz/robots-txt.html)) or with forms instead of links.

If the page still gets into the index, set the [canonical URL](https://developers.google.com/search/docs/crawling-indexing/consolidate-duplicate-urls) correctly so that the link value is used efficiently. Bad approaches, such as `nofollow` links, blocking in robots.txt, or inappropriate meta tags, can cause loss of link value or unnecessarily burden robots. It is important to [minimize errors and optimize](https://www.linkedin.com/feed/update/urn:li:activity:7263156396886577152/) links for truly relevant pages.
