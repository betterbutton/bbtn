# Dev stack
> A set of technologies, tools, and practices used for developing, deploying, and operating a website or application (front-end, back-end, databases, infrastructure).

*[Dev stack](https://en.wikipedia.org/wiki/Solution_stack)* (development stack, solution stack, technology stack, vývojová sada) is the complete set of technologies and practices used to develop, deploy, and operate a website or web application—from code running in the browser through servers and databases to delivery infrastructure and tools that manage changes. In practical terms: the dev stack defines what can be released quickly, what will collapse under load, how updates proceed, and how to trace the root causes of errors.

At first glance, today's web is divided. A large portion of the public web runs on long-established, CMS-oriented stacks—with WordPress and PHP dominating. Simultaneously, however, surveys among developers show that new projects are concentrating around application stacks built on JavaScript: Node.js, React, relational databases like PostgreSQL, and containerized deployments. This creates a gap between what is maintained long-term and what is built anew. Most real-world websites fall somewhere in between.

The true costs of a stack become apparent only after launch. Lifecycles and support are decisive. Technologies with clearly defined support periods—such as [LTS versions](https://www.reknisioweb.cz/p/long-term-support)—turn maintenance into a planned activity, not a constant firefighting of crises. Without them, technical debt grows silently but relentlessly.

Performance and accessibility set an insurmountable minimum. WCAG 2.2 defines basic expectations for accessibility, while Core Web Vitals translate performance into measurable, user-perceptible outcomes. Stacks that complicate semantic HTML, effective caching, or content delivery fail in both areas.

Security cannot be separated from the stack itself. A modern stack must account for common web risks, a secure development process, and the integrity of the software supply chain—from dependencies and plugins to build and deployment.

From a long-term operational perspective, a dev stack is not a question of novelty but of resilience. The best stacks do not age dramatically—they remain understandable, observable, repairable, and fast long after the initial excitement of their deployment has faded.
