# Pixel
> The smallest visible display unit on a screen.

A [pixel](https://cs.wikipedia.org/wiki/Pixel) ([pixel](https://developer.mozilla.org/en-US/docs/Glossary/Device_pixel)) is the fundamental building block of a digital image. While it may seem negligible, it plays a crucial role in how designers and developers create comprehensible and functional user interfaces across devices.

On modern screens, a pixel is no longer what it used to be. One so-called [CSS pixel](https://www.vzhurudolu.cz/prirucka/css-pixel) is a logical unit, not a physical one. One CSS pixel corresponds to 1/96 of an inch. Thanks to high-resolution displays like Retina or 4K, one CSS pixel can correspond to four or more hardware pixels. This principle ensures that an element designed to be 44 × 44 pixels will appear approximately the same size on different devices.

But that's not all. If you don't understand how pixels behave, you can easily create blurry icons, hard-to-read text, or broken layouts. However, those who understand pixels can create interfaces that are sharp, comprehensible, and fast. Whether it's fine-tuning icons to a grid or providing images at various resolutions as needed.

Developers should think in so-called [density-independent pixels](https://m2.material.io/design/layout/pixel-density.html#density-independence) — in units of *dp* — to make designs work across different device types. A density-independent pixel corresponds to 1/160 of an inch. Designers, in turn, should understand pixel density ([DPI](https://cs.wikipedia.org/wiki/DPI)) to align elements precisely and avoid visual errors.

Pixels are not just a technical detail. When approached correctly, users don't notice them — because everything works as it should.
