# Vibe coding
> Vibe coding is a method of software creation using artificial intelligence, where developers give instructions in natural language and the model generates code with minimal manual intervention.

[Vibe coding](https://uxdesign.cc/cracking-the-code-of-vibe-coding-124b9288e551) represents a new approach to programming, where the user uses common language to describe the desired outcome. Artificial intelligence then generates functional code based on this description. The term was coined by researcher [Andrej Karpathy](https://x.com/karpathy/status/1886192184808149383?lang=en) in early 2025. In his own words, it is not traditional programming: "I just see something, say it, run it, and it mostly works."

This approach is quickly gaining popularity because it opens the world of programming to people without technical education. Instead of learning the complex syntax and rules of programming languages, all you need is an idea and the ability to describe it understandably.

However, [Vibe coding](https://en.wikipedia.org/wiki/Vibe_coding) is not without risks. Code created by artificial intelligence may not be secure or properly structured. It can contain errors that the user may not recognize because they don't understand the code. It may be vulnerable to attacks or process data incorrectly. Experts therefore warn that vibe coding is more suitable for personal projects or prototypes than for production applications.

Despite these warnings, this method is rapidly gaining popularity. In some startups, artificial intelligence creates the majority of the source code. Development is thus faster, simpler, and more focused on the idea itself than on its technical implementation.

Vibe coding is changing the way people approach programming. Language is becoming a tool for creation. It is an exciting development that nevertheless requires a responsible approach.
