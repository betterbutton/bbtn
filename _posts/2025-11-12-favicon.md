# Favicon
> Favicons are a set of website icons. From the original 16 × 16 px symbol in browsers since 1999, they have evolved into an adaptive, maskable, and vector-based brand mark for websites and applications.

A [favicon](https://cs.wikipedia.org/wiki/Favicon) ([favicon](https://en.wikipedia.org/wiki/Favicon)) is a small icon displayed next to a website's name in browser tabs, bookmarks, and search results. It serves as a quick visual signal that helps people recognize familiar sites and easily return to them. Modern browsers, search engines, and mobile systems use it for a consistent brand appearance and clearer orientation.

[It originated in 1999](https://favicon.im/blog/favicon-evolution-from-pixels-to-powerhouses), when [Internet Explorer 5](https://en.wikipedia.org/wiki/Internet_Explorer_5) introduced it as a simple `favicon.ico` file placed in the website's root directory. Today, it involves a [whole set of icons](https://dev.to/mursalfk/the-ultimate-guide-to-adding-a-favicon-to-your-website-4anc) in SVG, PNG, and ICO formats, linked via HTML tags and a metadata [manifest](https://www.w3.org/TR/appmanifest/) file.

Favicons also appear on the phone's home screen in systems like [iOS](https://developer.apple.com/library/archive/documentation/AppleApplications/Reference/SafariWebContent/ConfiguringWebApplications/ConfiguringWebApplications.html) or [Android](https://web.dev/learn/pwa/web-app-manifest#icons), in [progressive web apps](https://developer.mozilla.org/en-US/docs/Web/Progressive_web_apps/Manifest/Reference/icons), and in pinned panels, with each environment [having its own requirements](https://evilmartians.com/chronicles/how-to-favicon-in-2021-six-files-that-fit-most-needs).

From a user experience perspective, the favicon has a surprisingly important function. It strengthens brand recognition, reduces the time needed to find a familiar website, and extends the brand's presence into the browser interface itself. It also matters for search engine optimization, as it can appear [in search results](https://developers.google.com/search/docs/appearance/favicon-in-search). A [poorly designed](https://css-tricks.com/svg-favicons-and-all-the-fun-things-we-can-do-with-them/) or missing icon looks unprofessional, while a clear and clean brand mark increases credibility.

Web creators should always use a scalable SVG file – ideally in a [light and dark mode version](https://css-tricks.com/svg-favicons-in-action/) – and fallback versions for older browsers, supplemented by a 180 × 180 px PNG for iOS and a [maskable icon for Android](https://web.dev/articles/maskable-icon). Tools like [RealFaviconGenerator](https://realfavicongenerator.net/) or [pwa‑‑asset‑‑generator](https://www.npmjs.com/package/pwa-asset-generator) simplify creation and testing.

The favicon is often overlooked, yet it is a crucial part of digital identity. In a few pixels, it combines trust, recognizability, and design discipline – and proves that even the smallest details have great power on the web.
