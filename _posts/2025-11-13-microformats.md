# Microformats
> A way to embed structured data directly into readable HTML using simple classes and attributes.

[Microformats](https://wiki.knihovna.cz/index.php/Mikroform%C3%A1t) ([microformats](https://microformats.org/)) are small, standardized patterns in HTML that allow content to be understandable by machines without changing its appearance for users. They enable developers to mark up information about people, organizations, events, or reviews using elements that HTML already offers. A short code snippet, such as [h-card](https://microformats.org/wiki/h-card) or [h-event](https://microformats.org/wiki/h-event), can transform ordinary text into data that browsers, search engines, or other tools can automatically process.

Microformats emerged in the early years of the 21st century when web enthusiasts like *[Tantek Çelik](https://tantek.com/)* and *[Kevin Marks](https://kevinmarks.com/)* sought a more practical path to the semantic web. Their main principle was *"people first, then machines"*. This ensures that the markup remains readable even if a parser ignores it. Each format – from h-card for contact details to h-entry for blog posts – uses familiar HTML attributes like class instead of adding new elements.

Later, so-called [microformats2](https://microformats.org/wiki/microformats2) standardized the naming (`h-`, `p-`, `u-`, `dt-`, `e-`) and simplified processing. Today, they form part of the foundation of the [IndieWeb](https://indieweb.org/microformats) movement, which allows independent websites to mutually exchange information about people, posts, or reactions.

For designers and developers, microformats represent proof that semantics and usability can go hand in hand. They add structure, improve accessibility, and [enrich search results](https://developers.google.com/search/docs/appearance/structured-data/intro-structured-data), without requiring scripts or hidden metadata. Although [JSON-LD](https://json-ld.org/) prevails today, microformats remain an understandable and open way to give content meaning.

Their value lies in simplicity – in the idea that carefully written HTML can simultaneously serve humans and machines. That is essentially the very essence of the original web.
