# Operational Dashboard for Product Teams
> 

We first encountered an operational dashboard ten years ago in the browser and homepage team at Seznam, which was product-led by [Lukáš Kovač](https://www.linkedin.com/in/lukaskovac/?originalSubdomain=cz). They had a large TV on a rolling stand displaying real-time data. The dashboard partly helped monitor that everything was running smoothly, partly served as feedback on product decisions, and motivated the team.

An operational dashboard is one of the most important tools for creators, administrators, and owners of websites and applications. It enables quick access to current information about website or app performance, identifies issues, and supports effective decision-making. We will focus on what metrics an operational dashboard should include, how to present them correctly, and when to display only key alerts.

## Key Elements of an Operational Dashboard for Websites

For daily monitoring of website performance, an operational dashboard should include data such as the number of current visitors, which helps monitor traffic in real time. Average page load speed is another important indicator, as website speed affects user satisfaction. Ideally, it should stay below three seconds.

Important indicators are also the [Core Web Vitals](https://web.dev/articles/vitals): LCP (Largest Contentful Paint), INP (Interaction to Next Paint), and CLS (Cumulative Layout Shift), which provide a detailed view of user experience. The [conversion rate](https://www.strafelda.cz/konverzni-pomer) allows you to track how many users completed a desired action, such as a purchase or registration. If you run marketing campaigns, you should monitor their performance, including the number of clicks, [click-through rate](https://www.strafelda.cz/ctr) (CTR), and revenue generated by individual channels.

On the other hand, some metrics should only be displayed if they exceed set thresholds or signal a problem. These include server errors ([HTTP 5xx](https://www.strafelda.cz/stavovy-kod)), a sudden increase in which can indicate a serious issue. Similarly, [404 errors](https://php.baraja.cz/stavove-http-kody), which show users non-existent pages, can negatively impact their experience.

Other alerts may include security incidents, such as unauthorized access attempts or [DDoS attacks](https://nukib.gov.cz/download/publikace/doporuceni/Doporuceni_DDoS.pdf), and significant slowdowns in page loading if the average load speed exceeds 3 seconds.

## Key Elements of an Operational Dashboard for Mobile Apps

For apps, it's important to track metrics that reflect performance and user behavior. Basics include the [number of active users](https://uxcam.com/glossary/dau-mau/), ([DAU/MAU](https://www.geckoboard.com/best-practice/kpi-examples/dau-mau-ratio/)), which provides an overview of daily and monthly user engagement.

Average session length shows how long users stay in the app during a single visit. The [Retention Rate](https://en.wikipedia.org/wiki/Retention_rate) tracks how many users return to the app after their first interaction, with values typically monitored after the first, seventh, and thirtieth day. Identifying the most used app features helps understand which parts provide the greatest value to users. The financial section will then cover daily revenue generated from in-app purchases and ads.

Just like with websites, for apps it's also important to display problems only when they occur. These include [Crash Rate](https://www.storyly.io/glossary/crash-rate), which is the percentage of sessions ending with an app crash. If this value exceeds two percent, immediate action is needed.

It's also important to monitor API response time, as backend service slowdowns above 500 ms should trigger an alert. Low adoption of a new app version can signal a problem if most users are still using an older version. Last but not least, it's important to identify suspicious activity, such as an unusually high number of failed login attempts.

## Incorporating Financial Metrics into the Dashboard

Financial indicators have their place on an operational dashboard if they are relevant for daily decision-making. Important financial metrics include revenue, which should be tracked daily or monthly, including its breakdown by sources such as ads, purchases, and subscriptions.

[Customer Acquisition Cost](https://www.wiener.cz/360-vypocet-nakladu-na-prvniho-zakaznika-v-praxi/) (CAC) shows how much it costs to acquire a new customer and is particularly important when monitoring current campaigns. [Customer Lifetime Value](https://krejta.cz/marketingovy-slovnik/clv/#:~:text=CLV%20je%20zkratka%20pro%20Customer,kter%C3%BDm%20m%C3%A1me%20vytvo%C5%99en%C3%BD%20dlouhodob%C3%BD%20vztah.) (LTV) represents the long-term revenue per customer and helps assess the effectiveness of marketing activities. Another key metric is ROAS ([Return on Advertising Spend](https://www.marketingppc.cz/ppc/co-je-to-roas/)), or return on ad investment, which allows for evaluating campaign performance.

## Dashboard Design Recommendations

When designing a dashboard, simplicity and clarity are key. Display only key metrics and use color coding, for example green for normal status and red for problems. Important metrics should be larger and displayed at the top, while alerts and less critical information can be shown below.

Set thresholds that trigger a visual or audible alert when issues arise, and use appropriate visualizations, such as simple graphs, numerical indicators, and heat maps for easy orientation.

An operational dashboard is an indispensable tool for managing websites and applications. With a well-designed dashboard, you can constantly monitor key metrics and respond to issues immediately. Customize the dashboard to your team's needs and use only those metrics that help you achieve your goals.

At the reception or near the boardroom, you can have a strategic dashboard displaying the fulfillment of long-term KPIs. In DevOps team rooms, a dashboard with information about server uptime, certificate validity, volume of tokens consumed for APIs, and similar data will be useful.

If you're looking for inspiration or specific tools for dashboard creation, try for example [Google Looker Studio](https://lookerstudio.google.com/), [Grafana](https://grafana.com/), or [Firebase Analytics](https://firebase.google.com/docs/analytics).🎄14
