# Cynefin Framework
> A sense-making framework that helps determine whether a web problem requires a procedure, expert analysis, an experiment, or a crisis intervention. Cy

The Cynefin ([Cynefin framework](https://en.wikipedia.org/wiki/Cynefin_framework)) framework is particularly useful when a team needs to decide how to respond before they start looking for solutions. Its core idea is simple. Not every problem is the same. And therefore, not every problem should be solved in the same way.

In practice, this means distinguishing whether a situation is clear, complicated, complex, chaotic, or simply confusing at the outset.

For websites and digital services, this is very practical. Routine maintenance, backups, and known fixes belong in the clear domain. There, procedures and automation work. Expert diagnosis of performance, accessibility, or migration issues belongs in the complicated domain. Navigation design, content strategy, or conversion optimization often fall into the complex domain because user behavior changes along with the system. Outages, security breaches, and cascading failures belong in the chaotic domain. There, stabilizing the situation is the first priority. Only then does detailed analysis make sense.

This is precisely why Cynefin is valuable. It is not a universal method. It is a framework that helps choose the appropriate approach. It protects teams from a common error—the assumption that every complex situation can be solved with more analysis and that there is always one correct solution.

Moreover, the current conception of the framework emphasizes that it is not a static model. Situations can move between domains. And the greatest risk is often false certainty.
