# How to Apply Signal Detection Theory in User Research
> 

![Hits, misses, correct rejections, and false alarms in a single 2x2 matrix, described below.](https://substack-post-media.s3.amazonaws.com/public/images/915de46b-4249-48ff-b27a-0f9b73b7bb1b_555x546.png "555x546")

Assessment of signal presence vs. its actual presence (image [from the original article](http://Posouzení%20přítomnosti%20signálu%20vs.%20realita%20jeho%20přítomnosti))

Signal Detection Theory offers a framework for understanding how users distinguish between valuable information and distracting elements. Lawton Pybus advises [on how to identify if your project involves signal presence decisions](http://Posouzení%20přítomnosti%20signálu%20vs.%20realita%20jeho%20přítomnosti), and how to incorporate Signal Detection Theory into research design.

> "Most UX research won't require the rigor of a lab experiment, but SDT can provide deeper insight into how users distinguish signals from noise in a design."\
> **— [Lawton Pybus](https://depth.drillbitlabs.com/p/signal-detection-and-ux-research)**

Beyond basic methods like usability testing, analyzing data using Signal Detection Theory is recommended. It categorizes decisions into four types (hits, misses, false alarms, correct rejections) and helps identify where and why users make errors. It can be applied at a conceptual level or through explicit measurement (e.g., hit rate and false alarm rate).
