# Linkodrome

**How did Edge become the preferred browser?** The Microsoft Edge browser is the [second most used browser](https://www.similarweb.com/browsers/worldwide/desktop/) on desktops right after Google Chrome. Although there is about a fifty percentage point gap between them, it is still surprising that people have come to like a browser from Microsoft specifically. Microsoft's previous browser, Internet Explorer, didn't exactly have a good reputation. Interestingly, Edge is genuinely popular and its usage cannot be simply attributed to it being pre-installed on Windows computers. Moreover, the DMA in the EU now forces us to choose our default browser.

> "I have Edge on my work Mac. Voluntarily and happily."\
> **— [Petr Staníček](https://www.threads.net/@pixycz/post/C-keWNcNuds?xmt)**

SEO expert Pavel Ungr [admitted to using Edge on social networks](https://www.threads.net/@pavelungr_seologer/post/C-kCjhnNcJD?xmt and in the comments, other internet specialists are joining him. Like many other browsers, Edge is built on the Chromium engine and can have Chrome extensions installed. However, Edge also offers features for user comfort that Chromium lacks.

One of these practical features are [workspaces](https://learn.microsoft.com/cs-cz/deployedge/microsoft-edge-workspaces), which allow you to quickly open, close, and share sets of web pages. This comes in handy when, for example, you are working on several projects at once and don't want to have dozens of tabs open.

Others are profiles. You know them from Chrome, but Edge allows you to set a rule for [opening a specific website under a chosen profile](https://blogs.windows.com/msedgedev/2020/04/30/automatic-profile-switching/#:~:text=to%20open%20with.-,Automatic%20profile%20switching,-(available%20in%20Microsoft). This is an interesting option if, for example, you want work matters to automatically open in your work profile. Michal Altair Valášek shows how you can use [automatic profile switching](https://www.youtube.com/watch?v=eqsmO1EAj8A) in Edge to limit tracking when you change your access to services that require third-party cookie consent.

The point of testing is to uncover bugs and bottlenecks. It's enough that you've discovered the problem; you don't need frequency statistics for it. Take advantage of the fact that you already know about the problem and focus your next testing on a different part of the tested design or the next step in the scenario.

Give yourself space between tests so you can quickly summarize your findings. Give the designer time to incorporate changes. Modify the design continuously during testing. This way, you can test more variants in a shorter time.
